Titanic

Nous avons dans un premier temps recherché des informations sur le naufrage et le bateau en lui-même dans le but de pouvoir réaliser un schéma entités-associations comprenant les classes des passagers, des informations sur les passagers (nom, prénom, âge, sexe etc.), les canaux de sauvetage, les navires venus récupérer les passagers des canaux de sauvetage etc.

Dans un second temps, nous avons créer la base de données en justifiant chaque relation puis nous l’avons implémentée. Pour chaque relation créer nous avons exprimé des contraintes afin que lors de l’exploitation de la base de données, les informations obtenues soient cohérentes, par exemple on a vérifié que l’âge de la catégorie enfant était bien inférieure à 12 ans ou encore que chaque personnel fût bien compris dans une liste de rôle.

Lors de l’implémentation, nous avons réalisé des tests afin de vérifier les contraintes exprimées auparavant. Pour créer la base de données, nous avons utilisé l’application DBeaver CE

Enfin, nous avons chacun exploité les données afin de répondre à des questions portant sur le sauvetage des passagers, on a utilisé différentes requêtes pour une même question afin de pouvoir varier les manières d’exploiter les données.
La problématique étant de comprendre pourquoi il y a eu que très peu de passagers sauvés, on a donc exploité et visualisé les données pour chaque sexe et pour chaque classe ainsi que pour chaque tranche d’âge.

La réalisation de ce projet était dans nos attentes, le projet a été réalisé en groupe de deux ainsi chaque étape à été réalisée en commun. Nous avons réussi à atteindre les objectifs demandés.

Bilan compétences :
• Concevoir et mettre en place une base de données à partir d’un cahier des charges client.
• Administrer une base de données, concevoir et réaliser des systèmes d’informations décisionnels
• Identifier les besoins métiers des clients et des utilisateurs
• Identifier ses aptitudes à travailler dans une équipe

Apprentissage critique :
• Mettre à jour et interroger une base de données relationnelle (en requêtes directes ou à travers une application)
• Concevoir une base de données relationnelle à partir d’un cahier des charges.
